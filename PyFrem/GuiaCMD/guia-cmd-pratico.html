Windows CMD 10 Exercícios Iniciante Modo Administrador

📌 O que é o CMD e por que aprender?

O Prompt de Comando (CMD) é uma interface de texto do Windows onde você conversa diretamente com o sistema operacional usando comandos escritos. Sem janelas, sem cliques — você digita uma instrução e o sistema executa.

Profissionais de TI usam o CMD diariamente para automatizar tarefas, gerenciar servidores, diagnosticar problemas de rede e configurar sistemas com muito mais velocidade do que pelo modo gráfico. Dominar o CMD é uma habilidade fundamental para qualquer técnico de informática.

⚡ Como abrir o CMD antes de começar

1

Pressione Win + R no teclado para abrir a janela "Executar"

2

Digite cmd no campo e pressione Enter

3

Para abrir como Administrador, clique com botão direito no CMD e escolha "Executar como administrador"

4

O cursor piscando após C:\Users\seu_cpf> significa que o CMD está pronto para receber comandos

Lista de Exercícios — Explicação Prática
01
Listando Diretórios e Arquivos
Listar o conteúdo de uma pasta
dir 💡 "dir" vem de directory (diretório)
O dir é o equivalente ao "olhar para dentro de uma pasta". Ele mostra o nome, tamanho e data de todos os arquivos e subpastas no local onde você está. É sempre o primeiro comando a usar quando você quer saber "o que tem aqui?".
Prompt de Comando
C:\>cd Windows
:: navega até a pasta Windows primeiro
C:\Windows>dir
:: lista todos os arquivos e subpastas de C:\Windows
📋 Sua Tarefa

Navegue até C:\Windows e execute o comando dir. Observe a lista exibida: cada linha é um arquivo ou pasta. A coluna à esquerda mostra a data, no meio o tamanho e à direita o nome.

02
Criando um Novo Diretório
Criar uma nova pasta no diretório atual
mkdir 💡 "mkdir" = make directory (criar diretório)
O mkdir cria uma pasta nova. É o equivalente a clicar com botão direito no Explorer e escolher "Nova pasta" — mas muito mais rápido. Você pode criar a pasta no local atual ou informar um caminho completo.
Prompt de Comando
C:\>mkdir C:\Users\%username%\Desktop\ExerciciosCMD
:: cria a pasta ExerciciosCMD na Área de Trabalho
📋 Sua Tarefa

Crie a pasta ExerciciosCMD na Área de Trabalho. Depois abra o Explorer para confirmar que a pasta apareceu. %username% é uma variável automática que representa seu nome de usuário no Windows — o CMD substitui ela sozinho.

03
Navegando entre Diretórios
Mover-se entre diretórios
cd 💡 "cd" = change directory (mudar diretório)
O cd é o comando de navegação — você "entra" e "sai" de pastas com ele. Use cd nome_da_pasta para entrar, cd .. para voltar um nível acima e cd \ para ir direto à raiz do disco.
Prompt de Comando
C:\>cd "Program Files"
:: entra na pasta Program Files (aspas por causa do espaço no nome)
C:\Program Files>cd \
:: volta direto para a raiz C:\
📋 Sua Tarefa

Saia de C:\, navegue até C:\Program Files e volte para C:\. Observe que o caminho exibido no prompt sempre muda conforme você navega — isso confirma onde você está no sistema.

04
Renomeando Arquivos
Alterar o nome de um arquivo existente
ren 💡 "ren" = rename (renomear)
O ren renomeia um arquivo. A sintaxe é ren nome_atual novo_nome. Atenção: o arquivo precisa existir na pasta onde você está navegando. Você não move o arquivo, apenas muda o nome.
Prompt de Comando
C:\Users\seu_cpf\ExerciciosCMD>ren arquivo1.txt arquivo_renomeado.txt
:: renomeia arquivo1.txt para arquivo_renomeado.txt
📋 Sua Tarefa

Antes de renomear, crie o arquivo de teste com o comando echo teste > arquivo1.txt enquanto estiver dentro da pasta ExerciciosCMD. Depois execute o ren e confirme com dir.

05
Copiando Arquivos
Copiar arquivos de um lugar para outro
copy 💡 O arquivo original permanece no lugar
O copy copia um arquivo para outro local, mantendo o original intacto. A sintaxe é copy origem destino. Se a pasta de destino não existir, o comando retornará erro — então crie-a antes com mkdir.
Prompt de Comando
C:\>mkdir C:\Users\%username%\Backup
:: cria a pasta Backup (se ainda não existir)
C:\>copy C:\Users\%username%\ExerciciosCMD\documento.txt C:\Users\%username%\Backup
:: copia o arquivo para a pasta Backup
📋 Sua Tarefa

Crie primeiro o arquivo documento.txt dentro de ExerciciosCMD com echo conteudo > documento.txt. Em seguida, crie a pasta Backup e execute o copy. Verifique com dir que o arquivo existe nos dois lugares.

06
Movendo Arquivos
Mover arquivos entre diretórios
move 💡 Diferente do copy: o arquivo sai da origem ⚠️ O original é removido
O move transfere um arquivo de uma pasta para outra — o arquivo some da origem e aparece no destino. É como recortar e colar no Explorer. Útil para organizar arquivos sem duplicá-los.
Prompt de Comando
C:\>move C:\Users\%username%\ExerciciosCMD\exemplo.txt C:\Users\%username%\Backup
:: move o arquivo (remove da origem, coloca no destino)
📋 Sua Tarefa

Crie o arquivo exemplo.txt dentro de ExerciciosCMD, depois mova-o para Backup. Confirme com dir nas duas pastas: o arquivo deve aparecer apenas em Backup após o comando.

07
Apagando Arquivos
Remover arquivos específicos
del ⚠️ Não vai para a Lixeira — é permanente!
O del apaga arquivos permanentemente, sem passar pela Lixeira do Windows. Use com cuidado! Confirme o nome exato do arquivo antes de executar. Se errar o nome, pode apagar o arquivo errado.
Prompt de Comando
C:\Users\seu_cpf\ExerciciosCMD>del arquivo_teste.txt
:: apaga o arquivo definitivamente (sem confirmação)
📋 Sua Tarefa

Crie o arquivo arquivo_teste.txt com echo teste > arquivo_teste.txt, confirme com dir que ele existe, e então apague com del. Rode dir novamente para confirmar que sumiu.

08
Removendo Diretórios
Deletar pastas
rmdir 💡 Atenção: a pasta precisa estar vazia ⚠️ Use /S para pastas com conteúdo
O rmdir (ou rd) remove pastas. Por padrão, só funciona em pastas vazias. Para deletar uma pasta com arquivos dentro, adicione a opção /S (que remove tudo recursivamente) e /Q para não pedir confirmação.
Prompt de Comando
C:\>mkdir C:\Users\%username%\Desktop\PastaTeste
:: cria a pasta para o exercício
C:\>rmdir C:\Users\%username%\Desktop\PastaTeste
:: remove a pasta vazia
📋 Sua Tarefa

Crie a pasta PastaTeste na Área de Trabalho com mkdir, confirme visualmente no Explorer, e então use o rmdir para deletá-la. Verifique que desapareceu.

09
Criando uma Estrutura de Pastas
Criar hierarquia de diretórios com um único comando
mkdir com / 💡 Cria pastas aninhadas automaticamente
O mkdir também aceita caminhos com subpastas. Ao escrever mkdir Projetos\Python, ele cria a pasta Projetos e, dentro dela, a subpasta Python — tudo de uma vez. Sem esse recurso, você teria que criar pasta por pasta.
Prompt de Comando
C:\>mkdir C:\Users\%username%\Projetos\Python
:: cria Projetos e Python (dentro de Projetos) de uma vez
📋 Sua Tarefa

Execute o comando acima e depois abra o Explorer em C:\Users\seu_cpf. Você deve ver a pasta Projetos com a subpasta Python dentro. Isso simula uma organização profissional de projetos de desenvolvimento.

10
Exibindo a Estrutura de Diretórios
Visualizar a hierarquia de pastas
tree 💡 Mostra a árvore de pastas visualmente
O tree exibe a estrutura de pastas em formato de "árvore", facilitando a visualização da hierarquia. Use tree /F para incluir também os arquivos. É muito útil para documentar ou inspecionar a organização de um projeto.
Prompt de Comando
C:\>tree C:\Users\%username%\Projetos
:: exibe a árvore de diretórios de Projetos
C:\>tree C:\Users\%username%\Projetos /F
:: /F inclui os arquivos na visualização
📋 Sua Tarefa

Execute o tree na pasta Projetos criada no exercício anterior. O resultado deve mostrar a subpasta Python aninhada. Experimente também com o parâmetro /F para ver todos os arquivos.
